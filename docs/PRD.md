# Product Requirements Document (PRD)

> **버전:** 1.0.0
> **최종 수정:** 2026-05-19
> **상태:** 작성 중

---

## 1. 프로젝트 개요

### 1.1 배경

Next.js 15 App Router + Supabase 기반의 인증 플랫폼 스타터. 이메일/비밀번호 인증, 보호된 라우트, 테마 전환 기능이 구현되어 있으며 이를 토대로 실제 서비스로 확장하는 것을 목표로 한다.

### 1.2 목적

- 안전하고 확장 가능한 사용자 인증 시스템 제공
- 인증 이후의 실질적인 서비스 기능(데이터 관리, 대시보드 등) 개발 기반 마련
- Supabase RLS 기반의 멀티테넌트 데이터 구조 확립

### 1.3 범위

| 포함                       | 제외                    |
| -------------------------- | ----------------------- |
| 이메일/비밀번호 인증       | 소셜 로그인 (현재)      |
| 보호된 라우트 및 세션 관리 | 모바일 앱               |
| 사용자 프로필 관리         | 결제 시스템 (현재)      |
| 데이터 CRUD 기능           | 실시간 협업 기능 (현재) |

## 2. 사용자 페르소나

### 2.1 일반 사용자 (End User)

- **특징:** 서비스 가입 후 보호된 콘텐츠를 이용하는 주 사용자
- **기술 수준:** 낮음 — 직관적인 UI 필요
- **주요 니즈:**
  - 빠르고 안전한 로그인/회원가입
  - 비밀번호 분실 시 셀프 복구
  - 개인 정보 관리

### 2.2 관리자 (Admin)

- **특징:** 사용자 계정 및 데이터를 관리하는 운영자
- **기술 수준:** 중간 이상
- **주요 니즈:**
  - 사용자 목록 조회 및 계정 상태 관리
  - 데이터 모니터링

## 3. 현재 구현 상태

### 3.1 인증 플로우 (완료)

| 기능            | 라우트                  | 구현 방식                             | 상태    |
| --------------- | ----------------------- | ------------------------------------- | ------- |
| 로그인          | `/auth/login`           | `supabase.auth.signInWithPassword`    | ✅ 완료 |
| 회원가입        | `/auth/sign-up`         | `supabase.auth.signUp` + 이메일 인증  | ✅ 완료 |
| 회원가입 완료   | `/auth/sign-up-success` | 이메일 인증 안내 페이지               | ✅ 완료 |
| 비밀번호 찾기   | `/auth/forgot-password` | `supabase.auth.resetPasswordForEmail` | ✅ 완료 |
| 비밀번호 변경   | `/auth/update-password` | `supabase.auth.updateUser`            | ✅ 완료 |
| 이메일 OTP 확인 | `/auth/confirm`         | `supabase.auth.verifyOtp`             | ✅ 완료 |
| 인증 오류       | `/auth/error`           | 에러 메시지 표시                      | ✅ 완료 |

### 3.2 라우트 보호 (완료)

- `/protected` — 로그인한 사용자만 접근 가능
- 미인증 시 `/auth/login`으로 리다이렉트
- 사용자 JWT Claims 표시 (개발 확인용)

### 3.3 데이터 연동 (부분 완료)

- `/instruments` — `instruments` 테이블 조회 (읽기 전용, 서버 컴포넌트)

### 3.4 UI/UX 기반 (완료)

- Tailwind CSS + shadcn/ui 컴포넌트 시스템
- 다크/라이트 테마 전환 (`next-themes`)
- 반응형 레이아웃

## 4. 기능 요구사항

### 4.1 사용자 인증 (Auth)

#### FR-AUTH-01: 이메일 인증 강화

- **현재:** 회원가입 시 이메일 발송, OTP 검증
- **요구사항:** 미인증 사용자의 보호 라우트 접근 시 재인증 안내
- **우선순위:** 높음

#### FR-AUTH-02: OAuth 소셜 로그인

- **요구사항:** Google, GitHub 계정으로 로그인/가입
- **Supabase 설정:** Auth Providers에서 활성화 필요
- **우선순위:** 중간

#### FR-AUTH-03: 세션 만료 처리

- **요구사항:** 세션 만료 시 사용자에게 알림 후 로그인 페이지로 이동
- **현재 상태:** 미구현 (에러 발생 시 단순 리다이렉트)
- **우선순위:** 높음

#### FR-AUTH-04: 계정 삭제

- **요구사항:** 사용자가 직접 계정 삭제 요청 가능
- **구현:** `supabase.auth.admin.deleteUser` (서버 액션)
- **우선순위:** 중간

---

### 4.2 사용자 프로필 (Profile)

#### FR-PROFILE-01: 프로필 정보 관리

- **요구사항:**
  - 닉네임, 아바타 이미지 등록/수정
  - `profiles` 테이블과 연동 (Auth 트리거로 자동 생성)
- **우선순위:** 높음

#### FR-PROFILE-02: 아바타 이미지 업로드

- **요구사항:** Supabase Storage에 이미지 업로드, 공개 URL 저장
- **제약:** 파일 크기 5MB 이하, jpg/png/webp 허용
- **우선순위:** 중간

---

### 4.3 대시보드 (Dashboard)

#### FR-DASH-01: 인증 후 대시보드

- **요구사항:** `/protected`를 실질적인 대시보드로 개편
- **현재:** JWT Claims 표시 (개발용) → 실제 사용자 데이터로 교체 필요
- **포함 내용:** 활동 요약, 빠른 액션, 알림
- **우선순위:** 높음

#### FR-DASH-02: 데이터 목록 페이지

- **요구사항:** `/instruments` 페이지를 완전한 CRUD 기능으로 확장
  - 목록 조회 (페이지네이션, 검색, 정렬)
  - 새 항목 추가 (폼 + 유효성 검사)
  - 수정 및 삭제
  - RLS 정책: 본인 데이터만 접근
- **우선순위:** 높음

---

### 4.4 보안 및 접근 제어

#### FR-SEC-01: Row Level Security (RLS)

- **요구사항:** 모든 테이블에 RLS 정책 적용
  - `instruments`: `user_id = auth.uid()` 조건 필수
  - `profiles`: 본인만 수정, 전체 읽기 허용
- **우선순위:** 매우 높음 (보안 필수)

#### FR-SEC-02: 입력 값 유효성 검사

- **요구사항:** React Hook Form + Zod 스키마 기반 클라이언트 검증
- **현재:** 기본 HTML 유효성만 적용됨
- **우선순위:** 높음

---

### 4.5 사용자 경험 (UX)

#### FR-UX-01: 로딩 상태 및 에러 처리 표준화

- **요구사항:**
  - 비동기 요청 시 일관된 로딩 인디케이터
  - 에러 메시지 국제화 (한국어 대응)
  - Toast 알림 컴포넌트 도입
- **우선순위:** 중간

#### FR-UX-02: 접근성 개선

- **요구사항:** WCAG 2.1 AA 수준 준수
  - 모든 폼 요소에 `aria-label` 적용
  - 키보드 내비게이션 지원
  - 색상 대비 비율 4.5:1 이상
- **우선순위:** 중간

## 5. 비기능 요구사항

### 5.1 성능

| 지표                   | 목표           |
| ---------------------- | -------------- |
| 초기 페이지 로드 (LCP) | ≤ 2.5초        |
| 인터랙션 응답 (INP)    | ≤ 200ms        |
| 레이아웃 안정성 (CLS)  | ≤ 0.1          |
| Lighthouse 점수        | ≥ 90 (전 항목) |

### 5.2 보안

- HTTPS 강제 적용 (Vercel 기본 제공)
- Supabase RLS 모든 테이블 활성화
- 환경변수 서버/클라이언트 분리 (`NEXT_PUBLIC_` 구분)
- `service_role` 키 절대 클라이언트 노출 금지
- XSS 방지: React 기본 이스케이핑 활용, dangerouslySetInnerHTML 금지

### 5.3 확장성

- Supabase Free Tier → Pro 전환 시 코드 변경 없음
- 컴포넌트 단위 분리로 기능 추가 용이
- Server Actions 기반 API 로직 (별도 API 라우트 최소화)

### 5.4 유지보수성

- TypeScript strict 모드 유지
- `any` 타입 사용 금지
- 컴포넌트당 단일 책임 원칙
